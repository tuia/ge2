CRUDficador versão 1.0

Este manual tem como função auxiliar na construção de novas telas dos softwares relacionados, com base nos padrões já estabelecidos, otimizando a usabilidade do produto final.

Seguir os padrões aqui apresentados é fundamental para manter o sistema conciso e usável. Este manual é dinâmico e deve ser atualizado pela equipe interna com os novos padrões estabelecidos.

Introdução

O sistema foi projetado, preferencialmente, para resoluções maiores do que 1280px de largura.

Os módulos foram desenvolvido utilizando o framework Twitter Bootstrap 3.3.2. Seus principais componentes e estilos estão descritos neste manual, mas você pode checar sua documentação para se aprofundar ou retirar dúvidas.

Além do bootstrap, foram utilizados outros recursos externos para complementar algumas funções dos módulos. Cheque na lista a seguir:

Framework HTML, CSS, and JS

Base CSS

Os elementos apresentados aqui fazem parte da estrutura básica das páginas. Você pode se aprofundar neste e em outros elementos através da documentação do bootstrap.

Estrutura básica

<!DOCTYPE html>
<html>
  <head>...</head>
  <body>
    <div class="container">
    <!--/CONTEÚDO DA PÁGINA-->
    </div>
  </body>

Para começar, utilize uma div com a classe .container ou .container-fluid (para vários tamanhos de tela) logo após a tag body. Assim, você cria uma margem que deixa a tela mais limpa e funcional.

Atenção! Não utilize o .container caso esteja desenvolvendo uma página a ser exibida dentro de uma janela flutuante

Grid

Um sistema de grid serve para auxiliar na diagramação do conteúdo da página, posicionando e alinhando os elementos de forma proporcinal.

Para isso, utiliza-se linhas (.row) e colunas (.col) de tamanhos variados. Algumas regras são necessárias para seu melhor funcionamento:

  • Para as dimensões ficarem corretas, quando se utilizar o grid diretamente na página, ele deve estar dentro de uma div com a classe .container ou .container-fluid.
  • Utiliza-se as colunas (rows) para criar grupos horizontais de colunas.
  • O conteúdo deve estar dentro das colunas, e apenas estas devem ser as filhas diretas (elemento um nível abaixo no DOM) das linhas (rows).
  • Uma linha (row) possui até 12 colunas. A largura da coluna é definida pelo número no final da classe col-xs-, sendo .col-xs-1 o menor tamanho e .col-xs-12 o maior.

Botões

Para se utilizar um botão, basta adicionar a classe .btn e as classes da cor e tamanhos desejados ao elemento a, button, input [type="button"] ou input [type="submit"].

Cores

As cores são indicadas através das seguintes classes:

Exemplos

Código

<!-- Botão padrão -->
<button type="button" class="btn btn-default">Default</button>

<!-- Fornece peso visual extra e identifica a ação principal em um conjunto de botões -->
<button type="button" class="btn btn-primary">Primary</button>

<!-- Indica uma ação bem-sucedida ou positiva -->
<button type="button" class="btn btn-success">Success</button>

<!-- Botão contextual para mensagens de alerta informativo -->
<button type="button" class="btn btn-info">Info</button>

<!-- Indica que se deve tomar cuidado com esta ação -->
<button type="button" class="btn btn-warning">Warning</button>

<!-- Indica uma ação perigosa ou potencialmente negativa -->
<button type="button" class="btn btn-danger">Danger</button>

<!-- Retira a ênfase do botão, fazendo com que pareça um link e mantendo seu comportamento -->
<button type="button" class="btn btn-link">Link</button>

Dica! Você também pode utilizar as classes .active e .disabled para os estados ativo e inativado.

Alertas

Alertas são pequenas janelas com o inuito de chamar a atenção do usuário com mesnagens de erro, alerta, sucesso ou dicas. Na validação de formulários, o alerta deve vir no início do formulário, logo abaixo do título e instruções, com um conteúdo que auxilie o usuário. Os alerts não possuem uma cor padrão, precisando de classes adicionais de acordo com seu objetivo.

Ao utilizar o alerta de erro ou sucesso, é importante sempre oferecer uma alternativa ao usuário, como tentar novamente fazer a ação com erro ou desfazer a ação concluída.

Alertas de informações ou dicas podem ser fechados para não poluir a tela.

Exemplos

Sucesso! O item foi adicionado com sucesso. desfazer
Dica! Você pode clicar e arrastar o elemento para a janela desejada.
Cuidado! As informações aqui podem estar divergentes, é melhor checar.
Erro! Não foi possível salvar as alterações. tentar novamente

Código

<!--/Sucesso-->
<div class="alert alert-success" role="alert">...</div>

<!--/Dica-->
<div class="alert alert-info alert-dismissible" role="alert">
  <button type="button" class="close" data-dismiss="alert" aria-label="Close"><span aria-hidden="true">&times;</span></button>
 ...
</div>

<!--/Alerta-->
<div class="alert alert-warning" role="alert">...</div>

<!--/Erro-->
<div class="alert alert-danger" role="alert">...</div>

Snackbars

Snackbars fornecem um feedback claro e leve sobre a ação, bem como uma opção de desfazer esta mesma ação ou performar outra. Exibem uma mensagem breve no rodapé à esquerda da página. Utilize toasts para:

  • Exibir feedback de sucesso, após realizar ação em janela flutuante (que se fecha automaticamente).
  • Exibir feedback de qualquer ação no mapa (GE).
  • Exibir informações do sistema que não têm importância ou são puramente técnicas

Para consultar o script, leia a documentação do Bootbox.

Exemplo

Exibir Toast

<div class="bb-alert alert alert-success">
  <strong>Sucesso!</strong> O item foi adicionado com sucesso. <a href="#">desfazer</a>
</div>

Abas

Abas devem ser utilizadas para diagramar conteúdos similiares em uma mesma página.

  • Não utilize muitas abas em uma mesma linha. Tente utilizar até no máximo 6.
  • Não utilize abas com conteúdos muito diferentes. Se tiver muito conteúdo, tente agrupar de forma diferente. Por exemplo, separe parte em uma janela de entidade e a outra parte em abas.
  • Agrupe conteúdos por assunto e que utilizar sua tela irá encontrar o que precisa muito mais fácil.

Exemplo

Tabelas

Existem vários tipos de tabelas que podem ser utilizados. Para todos os casos, a estrutura básica é a mesma, variando apenas as classes do elemento table

<table class="table">
    <thead>
      <tr>
        <th>...</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>...</td>
      </tr>
    </tbody>
  </table>

Dica! O elemento thead é utilizado para definir o cabelçalho da tabela, e para suas células, deve se utilizar a tag th.

O elemento tbody é utilizado para definir o corpo da tabela, e para suas células, deve se utilizar a tag td.

Existe também a tag tfoot, para definir o rodapé das tabelas. Seu uso é opcional.

Campo de texto com contador de caracteres

O exemplo a seguir limita o usuário a 30 caracteres e exibe um contador. É necessário colocar o número de caracteres máximo no atribulo maxLength, bem como ajustar os valores no código jQuery.

Exemplo

30

Código HTML

<div class="input-counter-container">
    <input type="text" class="form-control" placeholder="digite" maxlength="30">
    <div class="input-counter">30</div>
  </div>

Código jQuery

$('.input-counter-container .form-control').keyup( function() {
    var len = $(this).val().length;
      if (len >= 31) {
        $(this).val() = $(this).val().substring(0, 30);
      } else {
        $('.input-counter').text(30 - len);
        if (len > 24) { $('.input-counter').css('color', 'red'); }
        else { $('.input-counter').css('color', 'inherit'); }
      }
  });

Formulários

Para criação e validação de formulários, é necessário seguir alguns padrões. Veja os exemplos.

Para mais informações sobre estilização de formulários, confira a documentação do Bootstrap.


Formulário padrão

A foto deve ter até 2mb

  • Agrupe. Os grupos de formulários (.form-group) auxiliam na validação, mudando de formatação de acordo com o status (erro ou sucesso).
  • Coloque etiquetas. Quando possível, utilize tanto o label e o atributo placeholder para informar como preencher o campo. Quando o espaço for pouco, utilize apenas o placeholder.
  • Dê dicas. Um pequeno texto (.help-block) embaixo do campo de texto pode evitar muitos erros.
  • Padronize. Mantenha os botões conclusivos do formulário sempre à direita, sendo a ação princial sempre a mais à direita.
  • Evidencie o mais importante. Destaque o botão principal (.btn-primary) do secundário (.btn-default) com suas cores.

Formulário com erro

Ops! Impossível prosseguir. Favor preencher corretamente os seguintes campos:

campo obrigatório

A foto deve ter até 2mb

  • Não mude o alerta de lugar. O alerta funciona como introdução deve vir sempre no topo, logo após o título do formulário.
  • Explique o errro. Um breve texto (.help-block) embaixo do campo com erro é o suficiente para não deixar ninguém perdido.
  • Não seja técnico, e sim pessoal. Mensagens como "erro 505" não significam nada para o usuário final. Explique os erros educadamente de forma que um leigo entenderia.
<form class="well">

  <div class="alert alert-warning" role="alert"><strong>Ops!</strong> Impossível prosseguir. Favor preencher corretamente os seguintes campos:</div>

  <div class="form-group has-error">
    <label>Email</label>
    <input type="email" class="form-control" placeholder="seu email">
    <p class="help-block">campo obrigatório</p>
  </div>
  ...
  
</form> 

Formulário com sucesso

A foto deve ter até 2mb

Feedback de sucesso (na página sequente)

Sucesso! Seus dados foram alterados corretamente. desfazer
  • Não use janelas de confirmação. O usuário sabe o que faz. Perguntar "você tem certeza que deseja fazer isso?" pode ser incômodo e reduzir a produtividade.
  • Ofereça opção de desfazer. Se o usuário fizer algo sem querer, ele saberá exatamente como voltar atrás.
  • Não incomode o usuário. Faça o alerta de sucesso sumir após alguns segundos.
  • Em casos especiais, peça a senha. Se a ação do usuário for tão impactante que precise de confirmação, utilize um modal que peça a senha de administrador. Assim você evita erros de forma mais segura.

Janelas Flutuantes

Janelas flutuantes ou janelas de entidades (.entity-window) são elementos arrastáveis pela tela. Utilize as janelas flutuantes:

  • Quando o conteúdo principal da tela for muito importante (assim o conteúdo secundário não esconde o primário e pode ser fechado quando não for mais necessário)
  • Em um formulário longo, quando o usuário precisar buscar uma informação rapidamente e depois voltar.
  • Para se detalhar uma entidade. As janelas possuem formatação própria para elas - com cabeçalho, informações rápidas e rodapé de ações.

Exemplo

abrir janela flutuante

Botão

Para o botão chamar a janela, ele deve possuir a classe .show-window-btn e seu atributo href deve apontar para a id da janela que será exibida, nesse caso, #entity-window-example



<a href="#entity-window-example" class="btn show-window-btn">abrir janela flutuante</a>

Dica! Para organizar seu código, você pode deixar as janelas de entidade sempre no final, já que elas ficam escondidas. Assim, você deixa todas elas juntas e pode focar no conteúdo principal da página.

  <!--/JANELA FLUTUANTE-->
  <div class="entity-window" id="entity-window-example">
    <div class="entity-content">

      <!--/CABEÇALHO DA JANELA-->
      <div class="entity-window-header">
        <a href="#" class="bookmark-button hastooltip" title="adicionar aos favoritos"><i class="fa fa-bookmark-o"></i></a>
        <div class="pull-right">
          <a href="#" class="btn btn-xs btn-default hastooltip" data-title="destacar"><i class="fa fa-external-link"></i></a>
          <a href="#" class="btn btn-xs btn-default btn-close"><i class="fa fa-times"></i></a>
        </div>
      </div>

      <!--/CORPO DA JANELA-->
      <div class="entity-body">
        <!--/CONTEUDO-->

      </div>

    </div>

Entidades

Elementos da rede, como alimentadores, dispositivos e equipamentos, são chamados no módulo de entidades.

Símbolos de entidade

Para melhorar sua identificação, elementos com a classe .entity-symbol possuem uma formatação diferenciada, para conter um ícone e um id ou informação relevante, bom como a possibilidade de ter esta informação copiada no clique. O símbolos de entidades podem ser utilizados em dois tamanhos: default (40px x 40px) e .large (58px x 58px).

Exemplos

3564321 3564321
.entity-symbol .entity-symbol .large

Código


  <a href="#" class="entity-symbol">
    <i class="fa fa-car"></i>
    <span>3564321</span>
  </a>

  <a href="#" class="entity-symbol large">
    <i class="fa fa-car"></i>
    <span>3564321</span>
  </a>

Status

Os símbolos de entidade também se diferenciam de acordo com o estado em que a entidade se encontram, variando de cor.

Exemplos

3564321 3564321 7894321 7894321 7894321 7894321
- .pendente .designado .acionado .em-execução .finalizado

Código

<!--ENTITY-SYMBOL STATUS: EM EXECUÇÃO-->
  <a href="#" class="entity-symbol em-execucao">
    <i class="fa fa-car"></i>
    <span>3564321</span>
  </a>

Tooltips

Tooltips ou hints são balõezinhos de ajuda que são exibidos ao se repousar o mouse sobre o elemento com a classe .hastooltip. O texto a ser exibido é passado através do atributo title. As tooltips podem ser usadas em diversos elementos, como links, botões e imagens.

Exemplo

Repouse o mouse sobre mim!

Código



<a href="#" class="btn btn-primary hastooltip" title="exemplo de tooltip">Repouse o mouse sobre mim!</a>
